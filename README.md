# WJDR 同抵集结计时器

《无尽冬日》多集结同步到达辅助工具。输入一个统一到达时间，再录入每支队伍的行军时长，页面会自动反推出各队发出集结的准确时刻，并在出发前和发出时提供语音提醒。

## 运行方式

### 环境要求

- Node.js 22.13 或更高版本
- npm 10 或更高版本

### 首次运行

在 PowerShell 中执行：

```powershell
cd D:\CodeX\wjdr\wjdr-rally-timer
npm.cmd ci
npm.cmd run dev
```

终端显示本地地址后，用浏览器打开：

```text
http://localhost:3000
```

开发和生产启动命令均监听 `0.0.0.0:3000`，局域网设备可以通过
`http://服务器局域网IP:3000` 访问。

## 使用域名访问

复制 `.env.example` 为 `.env.local`，填写允许访问服务的域名：

```dotenv
ALLOWED_HOSTS=rally.burgerl.com
```

多个域名使用英文逗号分隔，只填写主机名，不要包含协议或端口。将域名的 DNS
解析到服务器 IP 后，重新运行 `npm.cmd run dev` 或 `npm.cmd run start`，即可通过：

```text
http://rally.burgerl.com:3000
```

如需使用标准的 `80/443` 端口和 HTTPS，请使用 Nginx 或 Caddy 将域名反向代理到
`127.0.0.1:3000`，并在代理服务器上配置 TLS 证书。

### 后续运行

依赖已经安装后，只需要：

```powershell
cd D:\CodeX\wjdr\wjdr-rally-timer
npm.cmd run dev
```

### 生产构建

```powershell
cd D:\CodeX\wjdr\wjdr-rally-timer
npm.cmd run build
```

构建完成后，部署产物位于 `dist` 目录。

这里使用 `npm.cmd` 是为了避开部分 Windows PowerShell 的脚本执行策略限制；在 CMD 或其他终端中也可以直接使用 `npm`。

## 使用流程

1. 设置所有集结需要抵达目标的统一时间。
   也可以使用 1 分钟、90 秒、2 分钟、3 分钟或 5 分钟快捷按钮。
2. 为每条集结填写名称和游戏内显示的行军时长。
3. 设置出发前语音提醒秒数，建议 5 秒。
4. 点击“开始同步计时”。
5. 页面进入提醒窗口时会播报一次“准备”，到点再次提醒“发出”；语音会提前
   0.3 秒触发以抵消播放延迟，发出后当前条目会自动结束。
6. 两条集结的发出间隔小于 5 秒时，主倒计时区域会分隔展示各自的准确倒计时，并在准备阶段合并播报队伍名称；同组已出发条目会保留显示，直到该组全部处理完。

## 现场使用建议

- 开始前先确认系统时间准确。
- 点击开始后把页面保持在前台，避免手机浏览器冻结后台计时。
- 第一次使用语音时，确认设备未静音、浏览器允许播放声音。
- 游戏内行军时长发生变化时，先停止计时，修改后再重新开始。
- 配置自动保存在当前浏览器中，刷新页面不会丢失。
